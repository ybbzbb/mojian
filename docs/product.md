# Product

## Product Overview

mojian（墨简）是一个**AI 辅助网文创作系统**，一句话定义为「三个 SOP + 一个执行器」。它把「写一本网文小说」完整地定义为依次执行三个标准作业程序（SOP）：①风格抽取 → ②圣经·大纲构建 → ③写作循环。

核心理念：**一致性交给代码，品味交给人，状态交给程序。** SOP 与提示词是本项目的核心资产（全部开源、明文、运行时加载、项目级可覆盖）；程序（Rust 核心 + CLI + 桌面客户端）只是 SOP 的执行器，负责管状态、管调度、管统计，让创作可复现、可中断恢复、可被社区改进。

方法论提炼自一部 94 章 / 32 万字网文小说的真实创作实践。

## Target Users

- 主要用户：**网文写手**——想用 AI 辅助、但要求可控可复现、保留人工品味判断的创作者
- 次要用户：**风格包 / 提示词贡献者**——为不同题材（都市/玄幻/历史…）贡献风格包与写作指南的社区参与者

## Current Features

> 当前处于**理念定型阶段**，仓库仅有 `README.md` 与 `LICENSE`，尚无可运行代码。以下为「已固化的设计资产」，非「已实现能力」。

- 设计总纲 RFC：`README.md` + GitHub issue #1，定义产品理念、分工原则、执行器架构、路线图
- 三个 SOP 的设计草案（issue 讨论中）：
  - SOP① 风格抽取（#5）：从 3-5 部参考小说客观提炼结构 → `creative-brief.md`（借鉴定位）+ `creative-vision.md`（创意愿景）
  - SOP② 圣经·大纲构建（#6）：定义「小说的基因」——圣经九件套 + 全书/分卷大纲；含 VOID 修订机制
  - SOP③ 写作循环（#7）：批次流水线（骨架 → 人审 → 正文 → 人审），每卷 1 道 + 每批 2 道人工关卡

## Non-Goals

- 不追求「全自动写作」——人工品味关卡是设计核心，不可被完全绕过
- 不做去除人工确认的「一键成书」模式
- 执行器不承担创意判断——客观一致性检查归程序，主观品味归人
- 素材库不存参考小说原文段落，只存结构化骨架（节拍/统计），以守版权边界
- 一期不做浏览器/云端多用户协作（本地单机执行为主）
