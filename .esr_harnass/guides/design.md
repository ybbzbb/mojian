# Guide: docs/design.md

本文件说明如何生成业务项目的 `docs/design.md`。

仅当项目包含实际 UI 界面时才需要本文件。

## 目标

输出当前项目的 UI 设计规范基线，供 frontend 相关 agent 读取。

## 必备内容

- 设计目标或视觉原则
- 颜色与排版基线
- 关键组件规范
- 布局或导航约束

## 写法要求

- 以当前已采用的视觉系统为准
- 写稳定规则，不写一次性页面稿说明
- 如果项目没有成型设计系统，可先写最小可执行规范

## 对 agent 的价值

- builder-agent 在前端任务中据此约束实现
- archivist-agent 在 UI 规范变化时据此归档

## 示例骨架

以下展示 `docs/design.md` 的典型结构：

```markdown
# Design

## 视觉原则

（2-3 条核心设计原则，例如：简洁优先、信息密度适中、移动优先）

## 颜色与排版

- 主色：#1677FF
- 辅色：#52C41A（成功）、#FF4D4F（危险）
- 正文字号：14px；标题字号：16px / 20px
- 字体：系统默认 sans-serif

## 关键组件规范

### 按钮

- 主操作：实心蓝色（`type="primary"`）
- 次操作：线框按钮（`type="default"`）
- 危险操作：红色（`danger`）

### 表单

- 标签位置：输入框左侧或上方（统一选一种）
- 错误提示：输入框下方，红色文字

### 表格

- 分页：默认 20 条/页，支持 10/20/50 切换
- 操作列：固定在右侧

## 布局约束

- 最大内容宽度：1200px
- 侧边导航宽度：240px（收起时 64px）
- 顶部导航高度：64px
```
